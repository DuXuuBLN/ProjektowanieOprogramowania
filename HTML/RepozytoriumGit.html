Podstawy repozytorium:




Repozytorium Git to wirtualna baza danych
twojego kodu, w którym zapisują się go wszelkie zmiany.
Dzięki git w wypadku potrzeby cofnięcia zmiany lub utraty kodu
możesz powrócić do stanu projektu jaki został wcześniej zapisany w Git.


Reporytoria Git dzielimy na 2 rodzaję:
- Zdalne - Takie które są zapisywane w siecii
(Dzielą się one na publiczne, takie do których każdy ma dostęp,
oraz prywatne, do których możesz zajrzeć tylko i wyłącznie ty.)

- Lokalne - Takie które znajdują się na twoim urządzeniu,
są one stale przenoszone do repozytorium zdalnego.

Tworzenie oraz działanie:




1. Za pomocą komendy init inicjujemy repozytorium w wybranym folderze.
2. Następnie używając komendy add dodajemy pliki do repozytorium.
3. Łączymy nasze repozytorium zdalne z lokalnym za pomocą remote add origin.
4. Wybieramy gałęź reporytorium zdanelngo do ktorej zmiany zostaną wysłane.
5. Po zmiankach w plikach korzystamy z commit'u by zapisać zmiany.
6. Wysyłamy zmiany do repozytorium zdalnego za pomocą push.

Ważne komendy terminala:


Git Init - Inicjuje folder w którym
wywołujemy komende jako repozytorium Git.

Git Clone - Tworzy kopie aktualnego repozytorium
zdalnego, na swoim urządzeniu (lokalnym repozytorium).

Git Fetch - W wypadku pracy zespołowej pobiera zmiany
wprowadzone przez inne osoby, ale nie zmienia ich w plikach.

Git Pull - Działa na zasadzie komendy "Git Fetch" z tym,
że wprowadza zmiany w pliki lokalne.

Git Status - Ujawnia aktualny status plików w repozytorium.

Git Add - Informuję Git'a o wprowadzeniu zmian w repozytorium lokalnym.

Git Commit - Wprowadza zmiany dodane komendą "Git Add"
do repozytorium zdalnego.

Git Push - Wysyła dane z lokalnego do zdalnego repozytorium.
Git Branch - Pozwala nam rozdzielać gałęzie naszego repozytorium zdalnego.
